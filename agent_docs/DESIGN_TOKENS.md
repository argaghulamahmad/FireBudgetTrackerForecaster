# Design Tokens

**Source of Truth:** `src/index.css` — the `@theme` block is the single authoritative registry for all custom tokens. Tailwind v4 auto-generates utility classes directly from these CSS variables. No separate config file exists or should be created.

---

## The Token Contract (3 Rules)

1. **Before writing any color, spacing, or typography value** — check the `@theme` block in `src/index.css` first.
2. **If a matching token exists** — use its Tailwind utility class (`bg-health-bg`, not `bg-[#F2F2F7]`).
3. **If no matching token exists** — define it in `src/index.css @theme` before using it anywhere, then document it in this file.

## Token Hierarchy

```
┌─────────────────────────────────────────────────────────┐
│  PRIMITIVE LAYER                                        │
│  Raw values — never referenced directly in components   │
│  e.g. #F2F2F7, #1C1C1E, #4F46E5                        │
└─────────────────┬───────────────────────────────────────┘
                  │ defined in
                  ▼
┌─────────────────────────────────────────────────────────┐
│  SEMANTIC LAYER  ← Gold Standard                        │
│  src/index.css  @theme { }                              │
│  Human-intent CSS variables:                            │
│  --color-health-bg      → "page background"             │
│  --color-health-text    → "primary readable content"    │
│  --color-health-separator → "dividers and borders"      │
└─────────────────┬───────────────────────────────────────┘
                  │ auto-generated by Tailwind v4 into
                  ▼
┌─────────────────────────────────────────────────────────┐
│  COMPONENT LAYER                                        │
│  Tailwind utility classes consumed in .tsx files        │
│  bg-health-bg  text-health-text  border-health-separator│
│  These are the ONLY values components should reference  │
└─────────────────────────────────────────────────────────┘
```

**Tailwind Primitives** (indigo-600, rose-500, emerald-500) sit alongside the custom semantic layer. They are fixed semantic aliases — `indigo-600` always means "brand action" in this codebase. Do not swap them for other Tailwind blue shades without a documented decision.

## The Gold Standard File

**`src/index.css` — `@theme` block**

```css
@theme {
  /* Typography */
  --font-sans:    "DM Sans",  ui-sans-serif, system-ui, sans-serif;
  --font-display: "Outfit",   ui-sans-serif, system-ui, sans-serif;

  /* Neutral Surface Palette (Apple Health system colors) */
  --color-health-bg:        #F2F2F7;   /* page background */
  --color-health-surface:   #FFFFFF;   /* card / panel surface */
  --color-health-text:      #1C1C1E;   /* primary text */
  --color-health-secondary: #8E8E93;   /* secondary / muted text */
  --color-health-tertiary:  #C7C7CC;   /* placeholder / disabled */
  --color-health-separator: #E5E5EA;   /* borders and dividers */
}
```

All other colors come from Tailwind's built-in primitive scale and carry fixed semantic meaning in this project (see Status Tokens below).

## Adding a New Token

1. Define the CSS variable in `src/index.css @theme`:
   ```css
   @theme {
     --color-health-NEW: #HEXVALUE;  /* human-readable description */
   }
   ```
2. Use it in components via its auto-generated Tailwind class: `bg-health-NEW` / `text-health-NEW`
3. Add a row to the **Semantic Token Reference** table in this file
4. Never import or reference the raw hex value in any `.tsx` file

---

## Theme Switching (Current State)

**Light mode only.** No dark mode or high-contrast mode is implemented. If dark mode is added in the future:

- Add a `dark:` variant for every `@theme` variable (e.g., `dark --color-health-bg: #1C1C1E`)
- Do **not** use hardcoded values — extend the semantic layer so every token has a dark variant
- Tailwind v4 supports CSS `@media (prefers-color-scheme: dark)` within `@theme` blocks
